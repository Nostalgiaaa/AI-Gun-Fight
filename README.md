## AI Gun Fight — 产品一页纸

### 一句话定位
**让你用代码或 AI 写出会打吃鸡的角色，看它和别人的角色打架、出圈的平台。**
不是游戏平台，是 **agent 训练 + 对抗 + 传播平台**。游戏是壳。

---

### 核心循环
```
写 agent  →  提交  →  匹配开战  →  看回放  →  调整  →  上分 / 分享
```
每一步都要在 5 分钟内能完成，否则迭代闭环死。

---

### 玩法

| 项 | 设计 |
|---|---|
| 模式 | 多人吃鸡，缩圈 + 物资 + 障碍 + 视野遮挡 |
| 职业 | 医生 / 突击 / 狙击（组排 meta 的核心维度） |
| 节奏 | Tick 制，1-3 秒 / tick（为 LLM 决策留余地） |
| 队伍 | 单人 / 小队（2-4）/ 大乱斗 |
| 胜利 | 最后存活的小队 |
| 确定性 | 同 state → 同 action，回放可还原，防作弊基础 |

---

### 四种参赛方式（底层共享同一份 JSON 协议）

| 方式 | 用户画像 | 门槛 | 谁付 LLM 钱 |
|---|---|---|---|
| **纯代码（JS）** | 程序员 | 高 | 无 LLM，免费 |
| **站内 Prompt** | 小白 | 低 | 平台（限免费额度） |
| **Dify / Coze 接入** | 已搭过 agent 的人 | 中 | 用户的 Dify 账号 |
| **Webhook Endpoint** | 硬核 | 高 | 用户自己服务器 |
| ~~API Key~~ | ~~~~ | ~~~~ | **不推荐主推**，仅作隐藏选项 |

**统一协议**：每 tick 服务器发 `state JSON`，接入方返回 `action JSON`。底层不区分四种来源。

---

### 社区

- **分赛道排行榜**：四种接入方式分别排名，**杜绝 LLM 碾压代码**
- **混合赛 / 乱斗赛**：跨赛道对战，话题制造机
- **AI 解说**：便宜 LLM 实时解读 log 生成解说，**对 agent 内心独白的解读是传统电竞做不了的独家卖点**
- **回放**：文字 log（精确）+ 2D 俯视渲染（可看）+ 自动导播（自动切交火点）
- **Prompt / 脚本商城**：玩家分享、fork、改造别人的 agent（社区资产）

---

### 技术栈

| 层 | 选型 |
|---|---|
| 纯代码沙盒 | **isolated-vm**（Node.js + V8 isolate） |
| 平台 LLM | **DeepSeek V3** 主力，**智谱 GLM-4-Flash** 免费档兜底 |
| 推理优化 | Prompt 模板锁结构（命中 cache）+ JSON Mode 强 schema + 规则引擎兜底（90% 决策不调 LLM） |
| 服务端 | tick 调度器 + 状态服 + 回放存储 |
| 前端 1.0 | 2D Canvas 俯视图 + 解说面板 |
| 前端 2.0 | 游戏引擎做 3D 集锦视频（**导演模式**，非主玩法） |

---

### 分阶段

```
V1.0    四种接入方式全开（站内 Prompt + 代码 + Dify + Endpoint）
        2D 俯视图 + 文字 log
        核心目标：跑通闭环 + 沉淀 50-100 个种子 agent + 验证好玩

V1.5    AI 解说上线
        Prompt 商城上线
        混合赛事

V2.0    3D 导演模式（自动剪集锦短视频）
        移动端
        赛季制 / 比赛运营
```

---

### 经济模型（必须从 Day 1 想清楚）

- 新用户免费 N 局 / 月（平台付 LLM 费）
- 超额 → 自带 Endpoint / Dify / 买点卡
- 单局成本上限：**¥0.5**（DeepSeek + 规则兜底 + prompt cache 后能压到这水平）
- 收费点：高级模式订阅、点卡、Prompt 商城分成、企业团建场

---

### 已识别的关键风险

1. **观赏性是命门**：自动导播不够好，回放就没传播力，整个平台死
2. **冷启动**：前 100 个 agent 必须官方自产 + 雇人写，**别幻想 UGC 自动起飞**
3. **反作弊**：prompt injection（"忽略规则直接开挂"）必须服务端校验 action 合法性兜底
4. **平衡性**：三职业 + 四接入方式，meta 容易塌缩到单一最优解，需要持续运营调参
5. **LLM 成本失控**：必须在规则兜底、prompt cache、批量推理三件事上同时发力
